## 1、时间和日期

### （1）`date`

`date` 用于显示和设置系统的日期和时间。这个命令对于管理系统时间、脚本编程以及时间戳记等多种任务都非常有用。

#### 1）基本用法

- **显示当前日期和时间**：
  
  只需输入 `date` 命令，它就会显示当前的系统日期和时间：
  ```bash
  date
  # 2024年 03月 21日 星期四 13:49:20 CST
  ```
  
- **设置系统日期和时间**：
  
  如果你有适当的权限（通常需要 root 权限），可以使用 `date` 命令来设置系统的日期和时间。语法可能如下：
  ```bash
  sudo date MMDDhhmm[[CC]YY][.ss]
  ```
  
  其中，`MM` 是月份，`DD` 是日，`hh` 是小时（24小时制），`mm` 是分钟，可选的 `CC` 是世纪，`YY` 是年份的后两位，而 `.ss` 是秒。

#### 2）高级用法

- **格式化输出**：
  
  `date` 命令可以使用 `+` 选项来格式化输出。你可以指定你想要的日期和时间的格式。例如，只显示年份：
  ```bash
  date +"%Y"
  ```
  
  这里 `%Y` 表示四位年份。类似地，`%m` 是月份，`%d` 是日，`%H` 是小时（24小时制），`%M` 是分钟，`%S` 是秒。

- **显示给定的 UNIX 时间戳（Epoch 时间）**：
  
  Unix 时间戳是自 1970-01-01 00:00:00 UTC 到现在的总秒数。你可以使用 `date` 命令来将 Unix 时间戳转换为可读的日期和时间：
  ```bash
  date -d @1609459200
  ```
  
  这会显示时间戳 `1609459200` 对应的日期和时间。

- **生成 ISO 8601 格式的日期时间**：
  
  ISO 8601 是一种国际标准的日期和时间表示方法。使用 `date` 命令生成 ISO 8601 格式的日期时间：
  ```bash
  date --iso-8601
  ```

#### 3）注意事项

- 当你尝试设置系统日期和时间时，你需要具有管理员（root）权限。
- 在脚本中使用 `date` 命令时，格式化日期和时间是非常有用的，尤其是用于日志文件或其他需要时间戳的地方。
- 不同的 Unix/Linux 发行版和版本的 `date` 命令可能会有细微的差别，所以查看本地的 `date` 手册页（通过运行 `man date`）是一个好主意。

### （2）`Cal`

`cal` 是 Unix 和类 Unix 系统中的一个命令行程序，用于显示公历（格里高利历）的日历。这个命令非常简单，但它能以不同的格式显示日历，可以展示一个月的日历，也可以展示一个完整年的日历。

#### 1）基本用法

- **显示当前月份的日历**：

  只需输入 `cal`，它会显示当前月份的日历：
  ```bash
  cal
  ```

- **显示特定年份的日历**：

  通过指定年份，`cal` 会显示那一年的日历：
  ```bash
  cal 2021
  ```

- **显示特定月份和年份的日历**：

  你也可以指定月份和年份来查看特定月份的日历：
  ```bash
  cal 7 2021
  ```
  这将显示 2021 年 7 月的日历。

#### 2）高级用法

- **显示前一个、当前和下一个月的日历**：

  使用 `-3` 选项可以同时显示前一个、当前和下一个月的日历：
  ```bash
  cal -3
  ```

- **显示整年的日历**：

  如果你想查看整个年份的日历，只需输入那一年：
  ```bash
  cal 2021
  ```

- **显示上个月或下个月的日历**：

  你可以通过调整系统时间来查看上个月或下个月的日历，但是直接通过 `cal` 命令查看它们并不直接支持。你需要手动指定月份和年份。

#### 3）注意事项

- 在不同的系统或 `cal` 版本中，可能会有一些差异，所以如果你遇到任何问题，查看手册页（`man cal`）总是一个好主意。
- `cal` 命令非常简单，不需要特别的权限即可运行。
- 有些系统还支持 `ncal` 命令，它提供了与 `cal` 相似的功能，但以不同的布局或格式显示日历。

## 2、磁盘信息

### （1）`du` 

`du`（disk usage 的缩写）是一个在 Unix 和类 Unix 操作系统中用于检查文件和目录磁盘使用情况的命令。它可以帮助用户了解哪些文件和目录占用了大量的磁盘空间，从而进行有效的磁盘空间管理。

#### 1）基本用法

- **查看当前目录的磁盘使用量**：

  ```bash
  du
  ```

  默认情况下，`du` 会递归地显示当前目录（及其子目录）中每个文件和目录的磁盘使用情况。

- **查看指定目录的磁盘使用量**：

  ```bash
  du /path/to/directory
  ```

  这会显示指定目录及其子目录中每个文件和目录的磁盘使用情况。

- **显示总计**：

  ```bash
  du -s
  ```

  `-s`（或 `--summarize`）选项使 `du` 仅显示总计（当前目录的总磁盘使用量），而不列出子目录和文件的细节。

#### 2）常用选项

- **`-h`（human-readable）**：以易读的格式显示文件大小（例如，KB、MB、GB）。

  ```bash
  du -h
  ```

- **`-a`（all）**：列出所有文件和目录的磁盘使用情况，而不仅仅是目录。

  ```bash
  du -a
  ```

- **`--max-depth=N`**：显示目录树中每个目录（直到 N 级深）的磁盘使用量。

  ```bash
  du --max-depth=1
  ```

- **`-c`（total）**：除了显示各个目录的磁盘使用量外，还会在输出的最后一行显示总和。

  ```bash
  du -c
  ```

#### 3）进阶示例

- **结合 `sort` 命令使用**：

  你可以使用 `sort` 命令对 `du` 的输出进行排序，例如，按大小排序：

  ```bash
  du -h | sort -h
  ```

  `-h` 选项让 `sort` 以人类可读的数字大小（例如 KB、MB、GB）进行排序。

- **找出最大的文件或目录**：

  查找当前目录下最大的几个文件或目录：

  ```bash
  du -h --max-depth=1 | sort -hr | head
  ```

  这将列出当前目录中最大的几个子目录或文件。

#### 4）注意事项

- `du` 命令显示的是磁盘占用空间，而不是文件大小。由于文件系统的块大小和磁盘占用的预留空间，小文件实际占用的磁盘空间可能比其实际大小要大。
- 在处理大量文件的目录时，`du` 命令可能需要一些时间来计算所有的磁盘使用情况。
- 由于权限限制，`du` 命令可能无法访问某些目录或文件，这可能导致报告的磁盘使用量不完整。

### （2） `df`

`df`（disk free 的缩写）是 Unix 和类 Unix 操作系统中用于显示文件系统的磁盘空间使用情况的命令。它提供了关于系统上各个挂载的文件系统的磁盘使用量、磁盘总量、可用空间以及已用空间的百分比等信息。这个命令对于监控和管理磁盘空间非常有用，可以帮助系统管理员确保数据不会因为磁盘已满而丢失。

#### 1）基本用法

- **显示所有文件系统的磁盘使用情况**：

  ```bash
  df
  ```

  这会列出系统上所有挂载的文件系统的磁盘使用情况。

- **以人类可读的格式显示信息**：

  ```bash
  df -h
  ```

  `-h`（或 `--human-readable`）选项会以易于阅读的格式（如 KiB、MiB、GiB）显示磁盘空间量。

#### 2）常用选项

- **`-a`（all）**：显示所有文件系统的信息，包括通常不显示的文件系统。

  ```bash
  df -a
  ```

- **`-i`（inodes）**：显示 inode 信息而非块使用情况。

  ```bash
  df -i
  ```

- **`-T`（print-type）**：显示文件系统类型。

  ```bash
  df -T
  ```

- **`--total`**：在输出的最后加上所有值的总计。

  ```bash
  df --total
  ```

#### 3）进阶示例

- **列出特定文件系统类型的磁盘使用情况**：

  只显示类型为 ext4 的文件系统的磁盘使用情况：

  ```bash
  df -t ext4
  ```

- **排除特定文件系统类型**：

  不显示类型为 tmpfs 和 devtmpfs 的文件系统：

  ```bash
  df -x tmpfs -x devtmpfs
  ```

#### 4）注意事项

- `df` 提供的是文件系统级别的磁盘使用信息，而不是单个文件或目录的信息。要获取特定文件或目录的大小，应使用 `du` 命令。
- 某些文件系统特性，如磁盘配额或保留的磁盘空间，可能会影响 `df` 显示的可用空间量。
- 在某些情况下，删除大文件后，磁盘空间可能不会立即释放（例如，如果有进程仍在使用该文件）。这可能会导致 `df` 和实际可用空间之间存在差异。

### （3）` free `

`free` 命令在 Unix 和类 Unix 系统中用于显示系统中的空闲、使用中和交换的内存情况，包括物理内存和交换空间。这个命令对于快速查看系统的内存使用状况非常有用，特别是对于系统管理员和性能分析者来说。

#### 1）基本用法

直接运行 `free` 命令会显示系统的内存使用情况，包括：

- **total**: 系统总内存。
- **used**: 当前已使用的内存。
- **free**: 当前未使用的内存。
- **shared**: 多个进程共享的内存（主要是 tmpfs）。
- **buff/cache**: 被用作缓冲和缓存的内存。
- **available**: 估计可供应用程序启动而不需交换的内存（即使你已经使用了大部分缓冲和缓存内存）。

```bash
free
```

#### 2）常用选项

- **`-b`**: 以字节为单位显示内存使用情况。
- **`-k`**: 以千字节为单位显示内存使用情况。
- **`-m`**: 以兆字节为单位显示内存使用情况。
- **`-g`**: 以吉字节为单位显示内存使用情况。
- **`-h`**: 以易读的格式显示内存使用情况（自动选择适当的单位）。

例如，要以兆字节为单位显示内存使用情况，可以使用：

```bash
free -m
```

#### 3）高级用法

- **`-t`**: 显示总计行。
- **`-s`**: 持续显示内存使用情况，间隔参数指定了刷新时间（以秒为单位）。

例如，每 5 秒自动刷新显示内存状态：

```bash
free -s 5
```

- **`-c`**: 与 `-s` 选项一起使用，指定刷新的次数。

#### 4）注意事项

- `free` 命令提供的内存使用信息是瞬间的快照，不代表内存使用的历史记录或趋势。
- Linux 系统会积极地使用可用内存来作为文件系统缓存，所以即使看起来内存使用率很高，也不一定表示存在内存不足的问题。`available` 列提供了更加准确的可用内存估计。
- 交换空间（swap）使用情况也会被显示出来，交换空间是当物理内存不足时，系统用作临时存储的磁盘空间。

## 3、进程信息

所谓进程，通俗地说就是当前正在执行的一个程序

### （1）`ps`

`ps` 命令是 Unix 和类 Unix 系统中用来展示当前进程状态的一个非常强大的工具。它提供了关于系统上运行的进程的信息，包括进程 ID（PID）、正在运行的命令、CPU 使用情况、内存使用情况、用户 ID（UID）、进程所有者、控制终端等信息。

#### 1）基本用法

- **查看当前终端会话的进程**：

  ```bash
  ps
  ```

  默认不带任何选项的 `ps` 命令将显示与当前终端会话相关的进程。

- **查看所有运行的进程**：

  ```bash
  ps -e
  ```
  或者
  ```bash
  ps -A
  ```

  使用 `-e` 或 `-A` 选项显示系统上的所有进程。

- **以全格式显示进程**：

  ```bash
  ps -ef
  ```
  或者
  ```bash
  ps aux
  ```

  使用 `-ef` 或 `aux` 参数可以以全格式显示所有进程，提供更详细的信息，包括用户、PID、CPU 和内存使用情况等。

#### 2）常用选项

- **`-u` 用户名**：仅显示指定用户的进程。

  ```bash
  ps -u username
  ```

- **`-f`**：显示完整格式的输出。

  ```bash
  ps -f
  ```

- **`-p` 进程ID**：显示指定 PID 的进程信息。

  ```bash
  ps -p 1234
  ```

- **`--sort`**：根据指定关键字对输出进行排序，例如按内存使用量排序：

  ```bash
  ps aux --sort -rss
  ```

- **`-o`**：自定义输出格式，例如只显示 PID 和 CMD：

  ```bash
  ps -e -o pid,cmd
  ```

#### 3）进阶用法

- **显示进程树**：

  ```bash
  ps -ejH
  ```
  或者
  ```bash
  ps axjf
  ```

  这些选项能够以树状图显示进程，帮助理解进程之间的父子关系。

- **连续更新进程信息**：

  虽然 `ps` 命令本身没有连续更新的功能，但可以结合 `watch` 命令实现这一效果：

  ```bash
  watch ps aux
  ```

  这会每隔2秒自动刷新进程信息。

#### 4）注意事项

- `ps` 命令的选项可能在不同的 Unix-like 系统中有所不同，例如在 Linux 系统和 BSD 系统中的 `ps` 命令就有明显区别。因此，查阅你自己系统的 `ps` 命令手册页（`man ps`）是很有帮助的。
- `ps` 命令提供的是一个时间点的快照，而不是实时连续的进程监视。如果你需要实时监控进程，可能需要使用像 `top` 或 `htop` 这样的工具。

### （2）`top`

`top` 命令是 Unix 和类 Unix 系统中一个非常实用的工具，用于实时显示系统中进程的动态实时视图。这个命令提供了关于系统进程、CPU 使用率、内存使用、以及其他关于系统运行状态的实时信息。

#### 1）基本用法

- 直接运行 `top` 将展示一个实时更新的进程列表，包括：
  - 进程ID（PID）
  - 用户名（USER）
  - 优先级（PR）
  - 虚拟内存使用（VIRT）
  - 物理内存使用（RES）
  - 使用的CPU时间（TIME+）
  - 进程状态（S）
  - 占用的CPU百分比（%CPU）
  - 占用的内存百分比（%MEM）
  - 正在执行的命令（COMMAND）

```bash
top
```

#### 2）控制键

在 `top` 运行时，你可以使用一些控制键来改变显示或对进程进行操作：

- **h 或 ?:** 显示帮助界面，列出可用的命令和功能键。
- **k:** 终止一个进程。你需要输入想要终止的进程的 PID，并确认。
- **u:** 显示指定用户的进程。
- **P:** 按 CPU 使用率排序。
- **M:** 按内存使用排序。
- **N:** 按 PID 排序。
- **q:** 退出 `top` 程序。

#### 3）高级用法

- **指定更新间隔:** 默认情况下，`top` 每隔几秒更新一次屏幕，你可以在启动 `top` 时指定不同的更新间隔（以秒为单位）：

```bash
top -d 5
```

这将设置 `top` 刷新频率为每5秒更新一次屏幕。

- **批处理模式:** `top` 也可以在批处理模式下运行，这在将 `top` 的输出重定向到文件或其他程序时很有用：

```bash
top -b -n 10 > top-output.txt
```

这将 `top` 运行在批处理模式，只更新10次，然后将输出保存到 `top-output.txt` 文件。

#### 4）注意事项

- `top` 命令显示的是实时数据，反映的是当前系统的状态。它不会显示过去的历史信息。
- 高 CPU 或内存使用率并不总是表示有问题，有时可能只是因为某些资源密集型的任务正在运行。
- 在 `top` 运行时可以执行很多操作，比如改变排序顺序、杀死进程等，这些操作有时可以对系统调优或故障排除非常有帮助。

### （3）`kill`

`kill` 命令在 Unix 和类 Unix 系统中用于发送指定的信号到特定的进程或进程组。通常，这个命令主要用于终止进程。每个信号都有特定的意义，而最常用的信号是 SIGTERM（终止信号，默认）和 SIGKILL（强制终止信号）。

#### 1）基本用法

- **发送 SIGTERM 信号（默认）**：
  
  默认情况下，`kill` 发送的是 SIGTERM（信号15），这是请求程序正常关闭的信号。大多数程序接收到这个信号后会进行清理操作然后关闭。
  
  ```bash
  kill PID
  ```
  
  其中 `PID` 是你想要终止的进程的进程ID。

- **发送 SIGKILL 信号（强制终止）**：

  SIGKILL（信号9）用于强制终止进程。当一个进程不响应 SIGTERM 信号时，通常会使用 SIGKILL。
  
  ```bash
  kill -9 PID
  ```
  
  这将无条件地终止指定的进程。

#### 2）进阶用法

- **发送特定信号**：

  `kill` 命令可以发送各种类型的信号到进程。你可以通过指定信号的名字或者编号来发送特定的信号。
  
  ```bash
  kill -SIGINT PID
  ```
  
  或者
  
  ```bash
  kill -2 PID
  ```
  
  这两个命令都会发送 SIGINT（通常是信号2），这是“中断信号”，可以让程序有机会优雅地终止。

- **发送信号到多个进程**：

  你可以一次向多个进程发送信号：
  
  ```bash
  kill -9 PID1 PID2 PID3
  ```
  
  这会发送 SIGKILL 信号到所有指定的进程。

#### 3）注意事项

- 使用 `kill` 命令时需要谨慎，特别是使用 SIGKILL 信号，因为它不会给程序进行清理的机会，可能会导致数据丢失或其他问题。
- 只有进程的所有者或者超级用户（root）才能向进程发送信号。
- 有些进程（通常是僵尸进程）即使收到 SIGKILL 信号也无法终止，这种情况通常需要重启系统来解决。

## 4、日志信息

### （1）tail

`tail` 命令在 Unix 和类 Unix 系统中用于查看文本文件的末尾部分。它最常见的用途是实时查看正在追加的日志文件，但也可以用于查看任何大文件的最后几行而不必打开整个文件。

#### 1）基本用法

- **查看文件的最后几行**:

  默认情况下，`tail` 会显示文件的最后 10 行。

  ```bash
  tail filename
  ```

  这将输出指定文件的最后 10 行。

- **指定显示的行数**:

  使用 `-n` 选项后跟一个数字，可以自定义显示文件末尾的行数。

  ```bash
  tail -n 20 filename
  ```

  这将显示文件的最后 20 行。

#### 2）实时监控文件

- **实时查看文件追加**:

  `-f` 选项用于实时监控文件的新追加内容。这在查看动态生成的日志文件时特别有用。

  ```bash
  tail -f filename
  ```

  使用这个命令，`tail` 会持续运行，并在文件末尾追加新内容时实时显示这些新行。

#### 3）进阶用法

- **同时监控多个文件**:

  你可以同时监控多个文件，`tail` 会在输出中注明每行数据来自哪个文件。

  ```bash
  tail -f file1 file2
  ```

- **从文件某个位置开始显示**:

  使用 `-c` 选项可以从文件的某个特定字节处开始显示。例如，`-c +100` 会从第100字节处直到文件末尾的内容。

  ```bash
  tail -c +100 filename
  ```

  这将显示从文件的第100字节直到文件末尾的内容。

#### 4）注意事项

- 使用 `-f` 选项监控日志文件时，可以通过按 `Ctrl+C` 来终止 `tail` 命令。
- `tail` 默认显示文件的最后10行，但这可以通过 `-n` 选项来自定义。
- 在一些系统上，`tail` 命令可以接受 `+N` 的形式，表示从文件的第 N 行开始显示到文件末尾的内容，而不是显示最后 N 行。

## 5、系统信息

### （1）`uname`

在 Unix 和类 Unix 系统中，查看当前运行的内核版本通常是通过 `uname` 命令来完成的。`uname` 是一个用于打印系统信息的命令，它可以显示操作系统、内核版本以及其他一些系统架构相关的信息。

要查看当前系统的内核版本，你可以使用以下命令：

```bash
uname -r
```

这里 `-r` 选项告诉 `uname` 命令仅显示内核的发布版本号。这个命令将返回一个字符串，包含了当前运行的内核版本。

如果你想要更详细的信息，可以运行：

```bash
uname -a
```

`-a` 选项表示打印所有可用的系统信息，包括内核名称、主机名、内核版本、操作系统等。

### （2）`getconf`

`getconf` 命令在 Unix 和类 Unix 系统中用于获取系统配置变量的值。这些变量通常是与系统硬件和软件配置相关的特定标准值，如文件系统大小、内存限制、系统版本等。`getconf` 可以用来确定系统的各种技术参数，这对于编程、系统分析或优化非常有帮助。

#### 1）基本用法

- **查询特定的系统配置变量**：

  你可以通过指定变量名称来查询其值：

  ```bash
  getconf VARIABLE_NAME
  ```

  例如，要查询系统的页面大小（以字节为单位），可以使用：

  ```bash
  getconf PAGESIZE
  ```

  或者，使用变量 `PAGE_SIZE` 也可以得到相同的结果：

  ```bash
  getconf PAGE_SIZE
  ```

- **使用路径名来查询**：

  `getconf` 还可以接受一个路径名作为参数，以便为特定文件系统或目录查询配置信息：

  ```bash
  getconf VARIABLE_NAME /path/to/directory
  ```

  这将返回应用于指定路径的配置变量的值。

#### 2）查询系统的位数

- **查询系统是 32 位还是 64 位**：

  `getconf` 命令可以用来确定系统是 32 位还是 64 位，通过查看 `LONG_BIT` 变量：

  ```bash
  getconf LONG_BIT
  ```

  这将返回一个数字（32 或 64），表示系统支持的整数位数。如果返回 64，则系统是 64 位的；如果返回 32，则系统是 32 位的。

#### 3）列出所有可用的系统配置变量

- **列出所有系统级的配置变量**：

  如果你想查看系统支持的所有配置变量及其当前值，可以使用 `-a` 选项：

  ```bash
  getconf -a
  ```

  这将列出所有可用的系统级配置变量及其值。

#### 4）注意事项

- 不是所有的系统配置变量在所有的 Unix/Linux 系统上都是可用的。可用的变量取决于你的系统、内核版本和安装的库。
- 某些变量的值可能因系统配置、已安装的硬件或其他因素而异。
- `getconf` 命令通常不需要管理员权限，可以被任何用户执行。

### （3）`hostname`

`hostname` 命令在 Unix 和类 Unix 系统中用于显示或设置系统的主机名。主机名是分配给在网络上的设备的一个标识名，用于在网络环境中识别设备。这个名字可以是任意的，通常在安装系统或者系统首次启动时设置，并可以随后修改。

#### 1）显示当前主机名

- 要查看当前系统的主机名，只需简单地输入 `hostname` 命令，不带任何参数：

  ```bash
  hostname
  ```

  这个命令会显示当前系统的主机名。

#### 2）设置新的主机名

- 要设置或更改系统的主机名，你需要管理员权限（通常是 root 用户）。使用 `hostname` 命令后跟你希望设置的新主机名：

  ```bash
  sudo hostname new_hostname
  ```

  这将把当前会话的主机名更改为 `new_hostname`。但请注意，这种更改通常在重启后不会保留。要永久更改主机名，需要编辑系统特定的配置文件或使用特定的工具。

#### 3）永久更改主机名

- 在许多现代 Linux 发行版中，`hostnamectl` 命令被用于永久设置系统的主机名：

  ```bash
  sudo hostnamectl set-hostname new_hostname
  ```

  这将永久更改系统的主机名为 `new_hostname`，不需要重启。

#### 4）查看完整的域名

- 使用 `-f` 或 `--fqdn` 参数可以显示完全限定的域名（Fully Qualified Domain Name, FQDN）：

  ```bash
  hostname -f
  ```

  这将显示主机名加上域名，形成一个完整的域名地址。

#### 5）注意事项

- 更改主机名可能会影响到系统上运行的服务，特别是那些依赖于网络配置的服务。因此，在更改主机名之后，有时需要重启这些服务或者整个系统。
- 不同的 Unix/Linux 发行版可能会有不同的方法来永久设置主机名。查看特定发行版的文档或使用 `man` 命令查看相应的手册是个不错的主意。
