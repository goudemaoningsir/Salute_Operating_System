## 1、基础概念

### （1）网卡 

网卡（Network Interface Card，NIC）是连接计算机到局域网（LAN）或广域网（WAN）的硬件设备，负责将计算机中的数据转换成电信号，并通过网络线路传输到目标设备。每个网卡都有一个唯一的物理地址，称为 MAC 地址（Media Access Control Address），用于在网络中唯一标识该网卡。

### （2）IP 地址

IP 地址（Internet Protocol Address）是用于在网络中标识和定位设备的地址。它是一个由数字组成的标识符，用于唯一地识别网络上的每个设备。IP 地址通常分为两类：IPv4 地址和 IPv6 地址。

IPv4 地址采用 32 位二进制表示，通常以点分十进制的形式表示，例如：192.168.1.1。然而，随着互联网的发展和设备的增多，IPv4 地址资源日益枯竭，因此逐渐被 IPv6 地址所取代。

IPv6 地址是 IPv4 的下一代协议，采用 128 位二进制表示，通常以冒号分隔的十六进制表示法表示，例如：2001:0db8:85a3:0000:0000:8a2e:0370:7334。IPv6 地址的引入大大扩展了地址空间，以满足日益增长的网络设备需求。

IP 地址分为公网 IP 地址和私有 IP 地址。公网 IP 地址是由互联网服务提供商（ISP）分配的，用于在全球范围内唯一标识设备；而私有 IP 地址通常用于内部局域网（LAN）中，无法直接从互联网访问，但可以通过 NAT（Network Address Translation）技术实现访问外部网络。

IP 地址的唯一性和可路由性是构建互联网和实现网络通信的基础，它们决定了数据在网络中的传输路径和目标设备。

### （3）域名

域名（Domain Name）是互联网中用于标识和定位网络资源的字符集合。它通常由多个部分组成，以点号分隔，例如：example.com。域名的层次结构是从右向左逐级降低，最右侧的部分表示顶级域（Top-Level Domain，TLD），如.com、.net、.org等，而最左侧的部分则表示最具体的主机或子域。

域名的作用是将人类易于记忆的字符（如英文字母、数字、连字符等）与网络资源的 IP 地址进行对应，从而方便用户访问互联网上的网站、服务和其他资源。域名解析系统（Domain Name System，DNS）负责将域名解析为相应的 IP 地址，使用户能够通过域名访问到目标资源。

域名通常由注册商（Registrar）进行注册和管理，用户通过注册商购买和管理自己的域名。域名的注册费用通常按年支付，并提供了一系列管理工具，如域名解析、域名转移、域名续费等服务。

域名的选择对于网站和品牌的建立至关重要，一个好的域名应该简洁、易记、易拼写，并且能够准确地反映出网站的内容、品牌或主题。随着互联网的发展，越来越多的域名后缀被引入，如.com、.net、.org、.info、.biz等，以满足不同用户的需求和定位。

### （4）端口号

端口号（Port Number）是一种用于标识网络通信中不同应用程序或服务的编号。在计算机网络通信中，数据通过网络传输时，需要通过端口号来确定数据包应该被发送给哪个应用程序或服务。

端口号是一个 16 位的整数，取值范围为 0 到 65535。其中，0 到 1023 的端口号是系统保留端口号，用于标识一些常见的服务，如 HTTP 服务的端口号为 80，HTTPS 服务的端口号为 443，FTP 服务的端口号为 21 等。这些端口号通常被操作系统或网络服务占用，一般用户无法修改。

除了系统保留端口号外，其他端口号都可以用于自定义应用程序或服务。例如，Web 开发中常用的开发服务器默认使用的端口号为 8000 或 8080，数据库服务 MySQL 的默认端口号为 3306，SSH 远程登录服务的默认端口号为 22 等。

在网络通信中，数据包通过 IP 地址和端口号来识别目标主机和目标应用程序。IP 地址用于确定目标主机，而端口号则用于确定目标主机上的具体应用程序。通常情况下，数据包的目标地址由目标主机的 IP 地址和端口号组成，形式为 IP 地址:端口号。

端口号的选择通常取决于应用程序的设计和需要，但在实际应用中应避免使用系统保留端口号以及已经被其他应用程序占用的常用端口号，以避免端口冲突和应用程序无法正常运行的问题。

## 2、shutdown 

`shutdown` 命令用于关闭或重启计算机，它允许系统管理员安全地关闭系统，并向系统用户发送警告和通知消息。

### （1）基本用法

```bash
shutdown [选项] [时间] [警告消息]
```

### （2）参数说明

- `时间`：表示计划执行关机或重启的时间。可以是以下几种格式之一：
  - `now`：立即执行关机或重启。
  - `+m`：在 `m` 分钟后执行关机或重启。
  - `hh:mm`：在指定的小时和分钟执行关机或重启，例如 `21:30` 表示在 21:30 关机或重启。
- `警告消息`：可选参数，用于向系统用户发送警告或通知消息。

### （3）常用选项

- `-r` 或 `--reboot`：重启系统。
- `-h` 或 `--halt`：关闭系统，但不要重新启动。
- `-c` 或 `--cancel`：取消预定的关机或重启任务。
- `-f` 或 `--force`：强制关闭系统，忽略正在运行的进程。

### （4）示例

立即关闭系统，并发送警告消息：

```bash
shutdown now "系统即将关闭，请保存工作并退出。"
```

在 10 分钟后重启系统：

```bash
shutdown +10 "系统将在 10 分钟后重启，请注意保存工作。"
```

在指定的时间（21:30）关闭系统：

```bash
shutdown 21:30 "系统将在晚上 21:30 关闭，请注意保存工作。"
```

取消预定的关机或重启任务：

```bash
shutdown -c
```

强制关闭系统，忽略正在运行的进程：

```bash
shutdown -h now
```

### （5）注意事项

- 执行 `shutdown` 命令通常需要管理员权限，因此可能需要使用 `sudo` 或作为超级用户运行。
- 关机或重启操作会中断正在运行的进程和任务，因此在执行前应确保所有的工作都已保存和结束。
- 发送警告消息可以提醒系统用户及时保存工作并退出。

## 3、 ifconfig 

`ifconfig` 命令用于配置和显示系统的网络接口信息，包括 IP 地址、MAC 地址、子网掩码等。

### （1）基本用法

```bash
ifconfig [接口名] [选项]
```

### （2）参数说明

- `接口名`：指定要配置或显示信息的网络接口名，如 `eth0`、`wlan0` 等。如果不指定接口名，则默认显示所有接口信息。
- `选项`：用于配置或显示特定信息的选项，如 `-a`、`-v` 等。

### （3）常用选项

- `-a`：显示所有接口信息，包括已停用的接口。
- `-v`：显示详细信息，包括接口的配置参数。
- `-s`：显示简要统计信息，包括接口的收发数据包数量。

### （4）示例

1. 显示所有接口信息：

   ```bash
   ifconfig
   ```

2. 显示指定接口（`eth0`）的详细信息：

   ```bash
   ifconfig eth0
   ```

3. 显示所有接口的详细信息：

   ```bash
   ifconfig -a
   ```

4. 显示接口的简要统计信息：

   ```bash
   ifconfig -s
   ```

### （5）注意事项

- 在使用 `ifconfig` 命令时，可能需要具有管理员权限，因此可能需要使用 `sudo` 或作为超级用户运行。
- 在最新的 Linux 发行版中，`ifconfig` 命令已经被弃用，推荐使用 `ip` 命令来替代。

## 4、 ip

`ip` 命令是 Linux 下用于配置网络接口和路由表的强大工具，它比传统的 `ifconfig` 命令更加灵活和功能强大。

### （1）基本用法

```bash
ip [选项] 命令 [对象]
```

### （2）参数说明

- `选项`：用于配置或显示特定信息的选项，如 `-s`、`-h` 等。
- `命令`：要执行的操作，如 `addr`、`link`、`route` 等。
- `对象`：操作的对象，可以是接口名、IP 地址、路由表条目等。

### （3）常用命令

**显示网络接口信息**

- `ip link show`：显示所有网络接口的信息，包括名称、状态、MAC 地址等。
- `ip addr show`：显示所有网络接口的 IP 地址信息，包括 IP 地址、子网掩码、广播地址等。

**配置网络接口**

- `ip link set <接口名> up/down`：启用或禁用指定的网络接口。
- `ip addr add <IP地址/子网掩码> dev <接口名>`：为指定接口添加 IP 地址。
- `ip addr del <IP地址/子网掩码> dev <接口名>`：从指定接口删除 IP 地址。

**配置路由表**

- `ip route show`：显示当前系统的路由表信息。
- `ip route add default via <网关地址>`：添加默认路由。
- `ip route add <目标网络/子网掩码> via <下一跳地址>`：添加指定网络的路由。

**其他常用命令**

- `ip neigh show`：显示 ARP 缓存表，即邻居表，包含了 IP 地址与 MAC 地址的对应关系。
- `ip tunnel`：管理网络隧道。
- `ip rule`：显示和配置路由策略。

### （4）示例

显示所有网络接口信息：

```bash
ip link show
```

启用或禁用指定网络接口：

```bash
ip link set eth0 up
ip link set eth0 down
```

添加默认路由：

```bash
ip route add default via 192.168.1.1
```

为指定接口添加 IP 地址：

```bash
ip addr add 192.168.1.100/24 dev eth0
```

### （5）注意事项

- `ip` 命令需要管理员权限才能执行某些操作，因此可能需要使用 `sudo` 或作为超级用户运行。
- `ip` 命令的语法相对复杂，但是功能更加丰富和灵活，是网络管理的重要工具。

## 5、ping 

`ping` 命令用于测试与另一台计算机的连通性，通过向目标计算机发送 ICMP 回显请求并等待回应来检查网络连接是否正常。

### （1）基本用法

```bash
ping [选项] 目标主机
```

### （2）参数说明

- `选项`：用于控制 `ping` 命令的行为，如 `-c`、`-i`、`-s` 等。
- `目标主机`：要 ping 的目标主机的域名或 IP 地址。

### （3）常用选项

- `-c count`：指定发送 ICMP 请求的次数。
- `-i interval`：指定发送 ICMP 请求的时间间隔。
- `-s packetsize`：指定发送的数据包大小。
- `-t timeout`：指定超时时间，单位为秒。
- `-q`：以安静模式运行，只显示结果，不显示统计信息。

### （4）示例

1. Ping 目标主机并连续发送 ICMP 请求，按 `Ctrl + C` 终止：

   ```bash
   ping example.com
   ```

2. Ping 目标主机并发送 5 个 ICMP 请求：

   ```bash
   ping -c 5 example.com
   ```

3. Ping 目标主机并设置超时时间为 2 秒：

   ```bash
   ping -t 2 example.com
   ```

### （5）注意事项

- 使用 `ping` 命令时需要有足够的网络权限，通常需要管理员权限或者是超级用户。
- `ping` 命令在网络故障排除和测试连通性时非常有用，但是在网络环境复杂或者受限的情况下可能会有误判。
- 在某些网络环境下，防火墙或路由器可能会屏蔽 ICMP 请求，导致 `ping` 不可用。

## 6、scp

`scp` 是一个用于在本地系统和远程系统之间进行文件传输的命令行工具，它基于 SSH（Secure Shell）协议进行安全的文件传输。

### （1）基本用法

```bash
scp [选项] [源文件] [目标地址]
```

### （2）参数说明

- `选项`：用于控制 `scp` 命令的行为，如 `-r`、`-P`、`-i` 等。
- `源文件`：要传输的文件或目录的路径。
- `目标地址`：目标主机上的文件路径，可以是本地文件系统路径或远程主机的地址，包括用户名、主机名和目标路径。

### （3）常用选项

- `-r`：递归复制整个目录。
- `-P port`：指定 SSH 连接的端口号。
- `-i identity_file`：指定用于身份验证的 SSH 私钥文件。
- `-p`：保留原文件的修改时间、权限和访问时间。
- `-q`：以安静模式运行，不显示传输进度信息。
- `-v`：显示详细的传输进度信息。

### （4）示例

从本地系统复制文件到远程系统：

```bash
scp file.txt user@remotehost:/path/to/destination/
```

从远程系统复制文件到本地系统：

```bash
scp user@remotehost:/path/to/file.txt /local/destination/
```

递归复制整个目录：

```bash
scp -r directory/ user@remotehost:/path/to/destination/
```

指定端口号和 SSH 私钥文件：

```bash
scp -P 2222 -i key.pem file.txt user@remotehost:/path/to/destination/
```

### （5）注意事项

- 使用 `scp` 命令进行文件传输需要目标系统具有 SSH 服务，并且目标用户具有相应的读写权限。
- 对于远程主机，需要确保目标路径的权限设置正确，以允许文件传输到该路径。
- 在进行文件传输时，建议使用 `-v` 参数显示详细的传输进度信息，以便及时了解传输状态。
